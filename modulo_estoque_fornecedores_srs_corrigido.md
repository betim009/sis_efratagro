# Módulo de Estoque e Fornecedores com Base no SRS Corrigido

## Solução

O módulo foi revisado para seguir o SRS corrigido como fonte de verdade. O estoque passa a ser controlado por produto e local em `estoques`, com toda movimentação registrada em `movimentacoes_estoque`. Entradas por compra exigem fornecedor ativo, local de destino e custo unitário. Saídas exigem local de origem e saldo suficiente. Transferências debitam origem e creditam destino dentro de transação. Ajustes registram motivo, usuário, data/hora e podem ocorrer sem fornecedor.

A integração com fornecedores foi ampliada para histórico real de compras via entradas de estoque, consulta de produtos fornecidos, suporte a fornecedor principal por produto e tabela para fornecedores secundários. A exclusão física de fornecedor foi protegida contra registros com histórico.

## Modelagem Revisada

- `locais_estoque`: mantém múltiplos locais com nome, descrição, tipo, código e status.
- `estoques`: saldo por `produto_id` + `local_estoque_id`; não salva estoque diretamente no produto.
- `movimentacoes_estoque`: agora registra fornecedor, cliente, custo unitário e valor total, além de tipo, produto, origem, destino, usuário, data/hora e motivo.
- `produtos.fornecedor_padrao_id`: representa o fornecedor principal do produto.
- `produtos_fornecedores`: permite fornecedores secundários e reforça no banco o limite de um fornecedor principal ativo por produto.

### Trecho de Migração Relacionado

```sql
-- ═══════════════════════════════════════════════════════════════════════
-- MÓDULO DE ESTOQUE/FORNECEDORES — ADEQUAÇÃO AO SRS CORRIGIDO
-- ═══════════════════════════════════════════════════════════════════════

ALTER TABLE movimentacoes_estoque ADD COLUMN IF NOT EXISTS fornecedor_id UUID REFERENCES fornecedores(id) ON DELETE RESTRICT;
ALTER TABLE movimentacoes_estoque ADD COLUMN IF NOT EXISTS cliente_id UUID REFERENCES clientes(id) ON DELETE SET NULL;
ALTER TABLE movimentacoes_estoque ADD COLUMN IF NOT EXISTS custo_unitario NUMERIC(14, 2);
ALTER TABLE movimentacoes_estoque ADD COLUMN IF NOT EXISTS valor_total NUMERIC(14, 2);

ALTER TABLE movimentacoes_estoque DROP CONSTRAINT IF EXISTS movimentacoes_estoque_tipo_movimentacao_check;
ALTER TABLE movimentacoes_estoque ADD CONSTRAINT movimentacoes_estoque_tipo_movimentacao_check CHECK (
  tipo_movimentacao IN ('ENTRADA', 'SAIDA', 'TRANSFERENCIA', 'AJUSTE')
);

ALTER TABLE movimentacoes_estoque DROP CONSTRAINT IF EXISTS movimentacoes_estoque_custo_unitario_check;
ALTER TABLE movimentacoes_estoque ADD CONSTRAINT movimentacoes_estoque_custo_unitario_check CHECK (
  custo_unitario IS NULL OR custo_unitario >= 0
);

ALTER TABLE movimentacoes_estoque DROP CONSTRAINT IF EXISTS movimentacoes_estoque_valor_total_check;
ALTER TABLE movimentacoes_estoque ADD CONSTRAINT movimentacoes_estoque_valor_total_check CHECK (
  valor_total IS NULL OR valor_total >= 0
);

CREATE TABLE IF NOT EXISTS produtos_fornecedores (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  public_id BIGINT GENERATED BY DEFAULT AS IDENTITY UNIQUE,
  produto_id UUID NOT NULL REFERENCES produtos(id) ON DELETE CASCADE,
  fornecedor_id UUID NOT NULL REFERENCES fornecedores(id) ON DELETE RESTRICT,
  principal BOOLEAN NOT NULL DEFAULT FALSE,
  status VARCHAR(20) NOT NULL DEFAULT 'ATIVO',
  criado_em TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  atualizado_em TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT uq_produtos_fornecedores UNIQUE (produto_id, fornecedor_id),
  CONSTRAINT produtos_fornecedores_status_check CHECK (status IN ('ATIVO', 'INATIVO'))
);

CREATE UNIQUE INDEX IF NOT EXISTS uq_produtos_fornecedores_principal
  ON produtos_fornecedores (produto_id)
  WHERE principal = TRUE AND status = 'ATIVO';

CREATE INDEX IF NOT EXISTS idx_produtos_fornecedores_produto_id ON produtos_fornecedores (produto_id);
CREATE INDEX IF NOT EXISTS idx_produtos_fornecedores_fornecedor_id ON produtos_fornecedores (fornecedor_id);
CREATE INDEX IF NOT EXISTS idx_movimentacoes_estoque_fornecedor_id ON movimentacoes_estoque (fornecedor_id);
CREATE INDEX IF NOT EXISTS idx_movimentacoes_estoque_cliente_id ON movimentacoes_estoque (cliente_id);
```

## Validações Implementadas

- Produto deve existir e estar ativo.
- Fornecedor é obrigatório em entrada por compra e deve estar ativo.
- Ajuste não exige fornecedor.
- Local deve existir e estar ativo quando usado.
- Quantidade deve ser maior que zero.
- Entrada por compra exige `custo_unitario > 0`.
- Saída e transferência não permitem saldo negativo.
- Transferência não permite origem e destino iguais.
- Ajuste exige motivo em `CORRECAO`, `CORRECAO_ERRO`, `ERRO`, `PERDA` ou `INVENTARIO`.
- Ajuste debita com `local_origem_id` ou credita com `local_destino_id`, nunca ambos.
- Compra com fornecedor inativo retorna erro de conflito.
- Exclusão de fornecedor com histórico retorna erro de conflito.

## Transações

Operações críticas usam `getClient()`, `BEGIN`, `COMMIT` e `ROLLBACK` no service:

- Entrada por compra: cria movimentação e credita saldo na mesma transação.
- Saída: bloqueia saldo com `FOR UPDATE`, valida disponibilidade, debita e registra movimentação.
- Transferência: bloqueia origem, debita origem, credita destino e registra movimentação.
- Ajuste: debita ou credita saldo e registra movimentação.

## Rastreabilidade

Toda movimentação registra:

- `tipo_movimentacao`
- `produto_id`
- `fornecedor_id` quando for compra
- `cliente_id` quando for saída para cliente
- `local_origem_id`
- `local_destino_id`
- `usuario_id`
- `data_movimentacao`
- `motivo`
- `observacoes`
- `custo_unitario` e `valor_total` quando for compra

Além da tabela de movimentações, o módulo registra auditoria operacional em `logs_auditoria`.

## Integração Futura com Financeiro

A entrada por compra já grava `fornecedor_id`, `custo_unitario` e `valor_total`, o que deixa o módulo pronto para gerar contas a pagar. A integração futura pode criar uma duplicata/conta a pagar após o commit da entrada, vinculando a movimentação de estoque ao documento financeiro de compra.

const listMovimentacoes = async ({

const AppError = require("./AppError");

const UUID_PATTERN =
  /^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i;
const SIMPLE_ID_PATTERN = /^\d+$/;
const EMAIL_PATTERN = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
const VALID_STATUS = ["ATIVO", "INATIVO"];
const VALID_TIPO_PESSOA = ["PF", "PJ"];

const onlyDigits = (value) => String(value || "").replace(/\D/g, "");

const sanitizeString = (value) => {
  if (value === undefined || value === null) {
    return null;
  }

  const sanitized = String(value).trim();
  return sanitized.length ? sanitized : null;
};

const validateUuid = (value, fieldName = "id") => {
  if (!UUID_PATTERN.test(String(value || ""))) {
    throw new AppError(`${fieldName} invalido`, 400);
  }
};

const validateEntityIdentifier = (value, fieldName = "id") => {
  const normalized = String(value || "").trim();

  if (!normalized) {
    throw new AppError(`${fieldName} invalido`, 400);
  }

  if (SIMPLE_ID_PATTERN.test(normalized) || UUID_PATTERN.test(normalized)) {
    return normalized;
  }

  throw new AppError(`${fieldName} invalido`, 400);
};

const validateEmail = (email) => {
  if (email && !EMAIL_PATTERN.test(email)) {
    throw new AppError("E-mail invalido", 400);
  }
};

const validateCpf = (cpf) => {
  const digits = onlyDigits(cpf);

  if (digits.length !== 11 || /^(\d)\1{10}$/.test(digits)) {
    return false;
  }

  let sum = 0;
  for (let i = 0; i < 9; i += 1) {
    sum += Number(digits[i]) * (10 - i);
  }

  let remainder = (sum * 10) % 11;
  if (remainder === 10) {
    remainder = 0;
  }

  if (remainder !== Number(digits[9])) {
    return false;
  }

  sum = 0;
  for (let i = 0; i < 10; i += 1) {
    sum += Number(digits[i]) * (11 - i);
  }

  remainder = (sum * 10) % 11;
  if (remainder === 10) {
    remainder = 0;
  }

  return remainder === Number(digits[10]);
};

const validateCnpj = (cnpj) => {
  const digits = onlyDigits(cnpj);

  if (digits.length !== 14 || /^(\d)\1{13}$/.test(digits)) {
    return false;
  }

  const calculateDigit = (base, factors) => {
    const sum = base.split("").reduce((accumulator, digit, index) => {
      return accumulator + Number(digit) * factors[index];
    }, 0);

    const remainder = sum % 11;
    return remainder < 2 ? 0 : 11 - remainder;
  };

  const firstFactor = [5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2];
  const secondFactor = [6, 5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2];

  const firstDigit = calculateDigit(digits.slice(0, 12), firstFactor);
  const secondDigit = calculateDigit(
    digits.slice(0, 12) + String(firstDigit),
    secondFactor
  );

  return digits.endsWith(`${firstDigit}${secondDigit}`);
};

const validateDocument = (document) => {
  const digits = onlyDigits(document);

  if (!digits) {
    throw new AppError("CNPJ/CPF obrigatorio", 400);
  }

  const isValid =
    (digits.length === 11 && validateCpf(digits)) ||
    (digits.length === 14 && validateCnpj(digits));

  if (!isValid) {
    throw new AppError("CNPJ/CPF invalido", 400);
  }

  return digits;
};

  const email = sanitizeString(payload.email);

  const status = sanitizeString(payload.status) || null;

      throw new AppError("Razao social obrigatoria", 400);
    }
  }

  if (!partial || payload.cnpj_cpf !== undefined) {
    if (!cnpjCpf) {
      throw new AppError("CNPJ/CPF obrigatorio", 400);
    }
  }

  validateEmail(email);

  }

  if (status && !VALID_STATUS.includes(status)) {
    throw new AppError("Status invalido para fornecedor", 400);
  }

  return {
    tipoPessoa: resolvedTipoPessoa,

    inscricaoEstadual: sanitizeString(payload.inscricao_estadual),

    complemento: endereco.complemento,
    bairro: endereco.bairro,
    cidade: endereco.cidade,
    estado: endereco.estado,
    observacoes: sanitizeString(payload.observacoes),

  };
