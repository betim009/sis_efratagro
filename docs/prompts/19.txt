PROMPT 19 — INTEGRAÇÃO FRONTEND + BACKEND (AXIOS, AUTH, ROTAS PROTEGIDAS E PERMISSÕES)

Você é um Engenheiro Full Stack Sênior especializado em React, Vite, Material UI, Axios, React Router, Node.js, Express e PostgreSQL para sistemas ERP profissionais.

CONTEXTO:
Estou construindo um ERP comercial completo com frontend em React + Vite e backend em Node.js + Express + PostgreSQL.
Os prompts anteriores já definiram:
- backend completo por módulos
- autenticação e autorização com JWT
- dashboard
- relatórios
- notificações
- auditoria
- frontend base
- telas principais do frontend

Agora preciso construir a camada de integração real entre frontend e backend, garantindo autenticação, controle de sessão, consumo padronizado da API, proteção de rotas e preparação para permissões granulares no frontend.

O QUE TENHO ATÉ AQUI:
- Base do frontend planejada
- AuthContext planejado
- Axios planejado
- ProtectedRoute planejado
- Backend com autenticação JWT e permissões granulares planejado
- Módulos principais já desenhados no frontend e backend

OBJETIVO:
Construir a camada completa de integração entre frontend e backend, com Axios configurado, interceptors, login real, persistência de sessão, logout, proteção de rotas, controle de permissões e tratamento padronizado de erros.

REGRAS:
- Use React com JavaScript
- Não use TypeScript
- Use Axios
- Use React Router
- Não criar integração bagunçada
- Não espalhar lógica de autenticação por vários componentes sem padrão
- Não acoplar diretamente toda chamada da API dentro das páginas
- A solução deve ser profissional, reutilizável e escalável
- O frontend deve ficar preparado para consumo real do backend
- Ao final, gere tudo em um documento markdown
