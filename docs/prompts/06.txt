PROMPT 06 — CRUD DE PRODUTOS

Você é um Engenheiro Backend Sênior especializado em Node.js, Express, PostgreSQL e arquitetura MSC para sistemas ERP.

CONTEXTO:
Estou construindo um ERP comercial com backend em Node.js + Express, banco PostgreSQL e arquitetura MSC.
O sistema já possui:
- estrutura inicial do backend
- modelagem do banco
- autenticação e autorização
- padrão de rotas
- módulo de fornecedores já construído
- módulo de clientes já construído

Agora preciso construir o módulo de Produtos, respeitando o SRS do projeto.

Segundo o SRS, o módulo de Produto deve permitir:
- cadastrar produto
- editar produto
- consultar produto
- inativar produto
- armazenar os campos principais do produto
- vincular fornecedor padrão
- controlar estoque mínimo
- controlar ponto de reposição
- manter código de barras ou referência interna
- permitir uso futuro no estoque, vendas e relatórios

Campos esperados no SRS:
- código
- nome
- descrição
- unidade de medida
- categoria
- preço de custo
- preço de venda
- peso
- código de barras / referência interna
- fornecedor padrão
- estoque mínimo
- ponto de reposição
- status

Status esperados:
- Ativo
- Inativo

O QUE TENHO ATÉ AQUI:
- Backend em Node.js + Express
- PostgreSQL
- Arquitetura MSC
- Middlewares de autenticação e autorização
- Banco já modelado
- Controle de permissões
- Estrutura de rotas pronta
- Módulos anteriores seguindo padrão profissional

OBJETIVO:
Construir o CRUD completo de Produtos no backend, de forma profissional, segura, validada e pronta para integração com estoque, vendas, alertas e relatórios.

REGRAS:
- Use JavaScript com Node.js e Express
- Não use TypeScript
- Respeite a arquitetura MSC
- Não colocar regra de negócio diretamente no controller
- Não fazer controller gordo
- Usar validações reais
- Não excluir produto fisicamente do banco
- Inativação deve ser lógica
- O módulo deve estar pronto para uso real
- O produto precisa ficar preparado para integração com estoque e vendas
- Toda rota deve respeitar autenticação e autorização
- Ao final, gere tudo em um documento markdown

ESCOPO DESTE PROMPT:
1. Criar produto
2. Listar produtos
3. Buscar produto por id
4. Atualizar produto
5. Inativar produto
6. Listar produtos por status
7. Listar produtos por categoria
8. Preparar consulta para saldo de estoque consolidado por produto
9. Preparar consulta para alertas de estoque mínimo e ponto de reposição

CAMPOS ESPERADOS:
- id
- codigo
- nome
- descricao
- unidade_medida
- categoria
- preco_custo
- preco_venda
- peso
- codigo_barras
- referencia_interna
- fornecedor_padrao_id
- estoque_minimo
- ponto_reposicao
- status
- created_at
- updated_at

VALIDAÇÕES IMPORTANTES:
- código obrigatório
- nome obrigatório
- validar duplicidade de código
- validar duplicidade de código de barras, se existir
- validar fornecedor_padrao_id quando enviado
- validar preco_custo como número válido
- validar preco_venda como número válido
- validar peso como número válido
- validar estoque_minimo como número válido
- validar ponto_reposicao como número válido
- validar status aceitando apenas:
  - ATIVO
  - INATIVO
- tratar produto inexistente corretamente
- impedir atualização inconsistente

REGRAS DE NEGÓCIO IMPORTANTES:
- produto inativo não deve aparecer normalmente em fluxos ativos, salvo consulta administrativa
- fornecedor padrão deve referenciar um fornecedor existente
- preço de venda não deve quebrar a estrutura mesmo que a regra comercial futura permita casos específicos
- produto deve ficar pronto para uso em estoque, movimentações, vendas e relatórios
- alertas de estoque mínimo e ponto de reposição devem considerar integração futura com a tabela de estoque
