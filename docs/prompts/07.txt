PROMPT 07 — MÓDULO DE ESTOQUE POR LOCAL E MOVIMENTAÇÕES

Você é um Engenheiro Backend Sênior especializado em Node.js, Express, PostgreSQL e arquitetura MSC para sistemas ERP com controle de estoque.

CONTEXTO:
Estou construindo um ERP comercial com backend em Node.js + Express, banco PostgreSQL e arquitetura MSC.
O sistema já possui:
- estrutura inicial do backend
- modelagem do banco
- autenticação e autorização
- módulos de fornecedores, clientes e produtos
- padrão profissional de rotas, services, models e controllers

Agora preciso construir o módulo de Estoque, respeitando o SRS do projeto.

Segundo o SRS, o módulo de Estoque deve permitir:
- cadastrar múltiplos locais de armazenamento
- controlar saldo de estoque por produto e por local
- registrar entradas de estoque
- registrar saídas de estoque
- registrar transferências entre locais
- manter rastreabilidade completa das movimentações
- notificar ou permitir consulta de produtos com estoque mínimo
- bloquear venda quando estoque for zero, de forma configurável no futuro

Também é exigido que toda movimentação tenha:
- data
- responsável
- motivo

O QUE TENHO ATÉ AQUI:
- Backend em Node.js + Express
- PostgreSQL
- Arquitetura MSC
- Middlewares de autenticação e autorização
- Banco já modelado
- Controle de permissões
- Produtos já cadastráveis
- Estrutura pronta para integração entre módulos

OBJETIVO:
Construir o módulo completo de Estoque por Local e Movimentações, de forma profissional, auditável, segura e pronta para uso real no ERP.

REGRAS:
- Use JavaScript com Node.js e Express
- Não use TypeScript
- Respeite a arquitetura MSC
- Não colocar regra de negócio diretamente no controller
- Não fazer controller gordo
- Usar validações reais
- Toda movimentação deve impactar corretamente o saldo do estoque
- Toda movimentação deve ser rastreável
- Não permitir saldo inconsistente
- Não permitir saída ou transferência acima do saldo disponível
- Toda rota deve respeitar autenticação e autorização
- O módulo deve ficar preparado para integração com vendas e alertas
- Ao final, gere tudo em um documento markdown

VALIDAÇÕES IMPORTANTES:
- nome do local obrigatório
- não permitir local duplicado com mesmo nome em contexto equivalente
- produto_id deve existir
- local_id deve existir
- quantidade deve ser maior que zero
- tipo_movimentacao deve aceitar apenas valores válidos
- motivo deve ser obrigatório nas movimentações
- usuario_id deve vir do usuário autenticado
- não permitir saída com saldo insuficiente
- não permitir transferência para o mesmo local
- tratar produto inexistente, local inexistente e estoque inexistente corretamente

REGRAS DE NEGÓCIO IMPORTANTES:
- entrada soma quantidade ao estoque
- saída reduz quantidade do estoque
- transferência gera duas movimentações coerentes:
  - saída no local de origem
  - entrada no local de destino
- o saldo final nunca pode ficar negativo
- o estoque deve ser criado automaticamente quando ainda não existir vínculo produto/local e a operação permitir isso
- movimentações devem ficar preparadas para integração com vendas e auditoria
- consulta de produtos abaixo do estoque mínimo deve considerar estoque consolidado ou por local, conforme você definir e justificar
- deixar preparada uma função ou service para futura validação de disponibilidade de estoque em vendas
