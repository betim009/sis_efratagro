PROMPT 18 — TELAS PRINCIPAIS DO FRONTEND (DASHBOARD, CLIENTES, PRODUTOS, ESTOQUE, VENDAS E FINANCEIRO)

Você é um Engenheiro Frontend Sênior especializado em React, Vite, Material UI, React Router e Axios para sistemas ERP administrativos profissionais.

CONTEXTO:
Estou construindo um ERP comercial completo com frontend em React + Vite e backend em Node.js + Express + PostgreSQL.
A base do frontend já foi planejada anteriormente com:
- estrutura de pastas
- layout principal
- sidebar
- header
- rotas públicas e privadas
- AuthContext
- Axios configurado
- tema com Material UI

Agora preciso construir as telas principais do sistema, respeitando a arquitetura já definida e preparando o frontend para integração real com a API.

Essas telas representam o núcleo operacional do ERP e serão as mais usadas no sistema.

O QUE TENHO ATÉ AQUI:
- Documento SRS definido
- Backend amplamente estruturado por prompts anteriores
- Frontend base já planejado
- Estrutura administrativa inicial já prevista
- Necessidade de telas com padrão profissional, reutilização e organização

OBJETIVO:
Construir o conjunto principal de páginas do frontend, com layout consistente, componentes reutilizáveis, estados de carregamento, filtros, tabelas, formulários e integração preparada com os serviços da API.

REGRAS:
- Use React com JavaScript
- Não use TypeScript
- Use Material UI
- Use Axios
- Use React Router
- Não criar páginas desorganizadas
- Não concentrar muita lógica em um único componente
- Separar bem páginas, componentes, dialogs, formulários e services
- A interface deve ter aparência profissional de sistema administrativo
- As páginas devem ser preparadas para integração real com backend
- O resultado deve ficar pronto para crescimento real
- Ao final, gere tudo em um documento markdown
