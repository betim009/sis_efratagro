PROMPT 10 — MÓDULO DE FROTA, MANUTENÇÃO E VINCULAÇÃO COM ENTREGAS

Você é um Engenheiro Backend Sênior especializado em Node.js, Express, PostgreSQL e arquitetura MSC para sistemas ERP com controle logístico e operacional de frota.

CONTEXTO:
Estou construindo um ERP comercial com backend em Node.js + Express, banco PostgreSQL e arquitetura MSC.
O sistema já possui:
- estrutura inicial do backend
- modelagem do banco
- autenticação e autorização
- módulos de fornecedores, clientes, produtos, estoque, vendas e financeiro
- padrão profissional de rotas, services, models e controllers

Agora preciso construir o módulo de Frota e Manutenção, respeitando o SRS do projeto.

Segundo o SRS, o módulo deve permitir:
- cadastrar veículos
- registrar tipo, placa, modelo, ano e responsável
- registrar manutenções preventivas e corretivas
- armazenar data, custo, fornecedor do serviço e descrição
- criar agenda de manutenção preventiva com alertas por quilometragem ou data
- manter histórico completo por veículo
- vincular veículo às entregas realizadas

Esse módulo deve ficar preparado para integração com entregas, dashboard e relatórios.

O QUE TENHO ATÉ AQUI:
- Backend em Node.js + Express
- PostgreSQL
- Arquitetura MSC
- Middlewares de autenticação e autorização
- Banco já modelado
- Controle de permissões
- Estrutura pronta para integração entre módulos
- Entregas já previstas na modelagem e no fluxo de vendas

OBJETIVO:
Construir o módulo completo de Frota e Manutenção, de forma profissional, auditável, segura e pronta para uso real no ERP.

REGRAS:
- Use JavaScript com Node.js e Express
- Não use TypeScript
- Respeite a arquitetura MSC
- Não colocar regra de negócio diretamente no controller
- Não fazer controller gordo
- Usar validações reais
- Toda manutenção deve ficar vinculada corretamente ao veículo
- Toda rota deve respeitar autenticação e autorização
- O módulo deve ficar preparado para integração com entregas, dashboard e relatórios
- Ao final, gere tudo em um documento markdown

ESCOPO DESTE PROMPT:
1. CRUD de veículos

VALIDAÇÕES IMPORTANTES:
- placa obrigatória
- não permitir placa duplicada
- modelo obrigatório
- ano deve ser válido
- tipo do veículo obrigatório
- responsavel_id, se enviado, deve existir
- status do veículo deve aceitar apenas valores válidos
- veiculo_id deve existir nas manutenções
- tipo_manutencao deve aceitar apenas:
  - PREVENTIVA
  - CORRETIVA
- status da manutenção deve aceitar apenas valores válidos
- custo deve ser número válido maior ou igual a zero
- fornecedor_id, se enviado, deve existir
- não permitir manutenção para veículo inexistente
- tratar veículo inexistente e manutenção inexistente corretamente

REGRAS DE NEGÓCIO IMPORTANTES:
- veículo pode existir sem manutenção
- manutenção sempre pertence a um veículo
- histórico do veículo deve reunir dados do cadastro e das manutenções
- veículo em manutenção pode ter status operacional diferente de ativo
- alertas preventivos devem poder considerar:
  - data agendada próxima
  - quilometragem de referência próxima ou atingida
- estrutura deve permitir vincular um veículo a uma entrega
- custos de manutenção devem ficar preparados para relatórios gerenciais
- ao concluir manutenção, o status do veículo pode voltar a ATIVO conforme regra definida e justificada
- o módulo deve ficar preparado para dashboard com veículos em manutenção
