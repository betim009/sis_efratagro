PROMPT 12 — DASHBOARD GERENCIAL COM MÉTRICAS, ALERTAS E CONSOLIDAÇÕES

Você é um Engenheiro Backend Sênior especializado em Node.js, Express, PostgreSQL e arquitetura MSC para sistemas ERP com foco em agregações, indicadores e painéis gerenciais.

CONTEXTO:
Estou construindo um ERP comercial com backend em Node.js + Express, banco PostgreSQL e arquitetura MSC.
O sistema já possui prompts anteriores cobrindo:
- estrutura inicial do backend
- modelagem do banco
- autenticação e autorização
- fornecedores
- clientes
- produtos
- estoque
- vendas
- financeiro
- frota
- entregas

Agora preciso construir o módulo de Dashboard, respeitando o SRS do projeto.

Segundo o SRS, o dashboard deve oferecer:
- painel principal com visão consolidada
- widgets informativos acessíveis por perfil
- filtros por período e unidade
- gráficos e consolidações para apoio à decisão

Indicadores esperados no SRS:
- saldo de estoque por produto / local
- vendas do dia / semana / mês
- duplicatas em aberto e vencidas
- pedidos de venda futura com prazo próximo
- alertas de estoque mínimo
- status da frota (veículos em manutenção)
- gráficos de desempenho de vendas no tempo

Esse módulo deve ficar preparado para ser consumido pelo frontend em React + Vite + Material UI.

O QUE TENHO ATÉ AQUI:
- Backend em Node.js + Express
- PostgreSQL
- Arquitetura MSC
- Middlewares de autenticação e autorização
- Banco já modelado
- Módulos de negócio principais já definidos
- Estrutura pronta para integração entre módulos
- Dados distribuídos entre estoque, vendas, financeiro, frota e entregas

OBJETIVO:
Construir o módulo completo de Dashboard gerencial, com endpoints consolidados, métricas operacionais e visão resumida do sistema, de forma profissional, segura, performática e pronta para uso real.

REGRAS:
- Use JavaScript com Node.js e Express
- Não use TypeScript
- Respeite a arquitetura MSC
- Não colocar regra de negócio diretamente no controller
- Não fazer controller gordo
- Usar queries e agregações coerentes com PostgreSQL
- O dashboard deve ser montado com foco em performance
- O módulo deve respeitar autenticação e autorização
- O resultado deve ficar preparado para filtros por período, unidade e perfil
- Sempre que fizer uma decisão importante de agregação, explique de forma curta e objetiva
- Ao final, gere tudo em um documento markdown

REGRAS DE NEGÓCIO IMPORTANTES:

- respostas devem ser estáveis e previsíveis para consumo do frontend
