PROMPT 13 — MÓDULO DE RELATÓRIOS GERENCIAIS COM EXPORTAÇÃO E FILTROS

Você é um Engenheiro Backend Sênior especializado em Node.js, Express, PostgreSQL e arquitetura MSC para sistemas ERP com foco em relatórios gerenciais, exportação de dados e consolidações por período.

CONTEXTO:
Estou construindo um ERP comercial com backend em Node.js + Express, banco PostgreSQL e arquitetura MSC.
O sistema já possui prompts anteriores cobrindo:
- estrutura inicial do backend
- modelagem do banco
- autenticação e autorização
- fornecedores
- clientes
- produtos
- estoque
- vendas
- financeiro
- frota
- entregas
- dashboard

Agora preciso construir o módulo de Relatórios, respeitando o SRS do projeto.

Segundo o SRS, o sistema deve permitir:
- geração de relatórios nos períodos:
  - diário
  - semanal
  - anual
- relatórios de:
  - vendas por período, vendedor e cliente
  - estoque atual e movimentações
  - duplicatas emitidas e recebidas
  - entregas realizadas e pendentes
  - manutenções de frota
  - vendas futuras
- exportação em PDF e Excel
- agendamento de relatórios por e-mail como funcionalidade opcional futura

Esse módulo deve ficar preparado para ser consumido pelo frontend e também para futuras automações.

O QUE TENHO ATÉ AQUI:
- Backend em Node.js + Express
- PostgreSQL
- Arquitetura MSC
- Middlewares de autenticação e autorização
- Banco já modelado
- Módulos operacionais já definidos
- Dashboard já estruturado
- Estrutura pronta para integração entre módulos

OBJETIVO:
Construir o módulo completo de Relatórios gerenciais, com consultas filtráveis, respostas consistentes, exportação e base preparada para crescimento futuro.

REGRAS:
- Use JavaScript com Node.js e Express
- Não use TypeScript
- Respeite a arquitetura MSC
- Não colocar regra de negócio diretamente no controller
- Não fazer controller gordo
- Usar queries coerentes com PostgreSQL
- O módulo deve respeitar autenticação e autorização
- Os relatórios devem aceitar filtros reais
- As respostas precisam ser consistentes tanto para visualização quanto exportação
- Sempre que fizer decisão importante de agregação ou exportação, explique de forma curta e objetiva
- Ao final, gere tudo em um documento markdown
